# Reguli de numire fișiere

## Format recomandat

Folosește numere la început pentru ordine:

- `00_MASTER_CONTEXT.md`
- `01_PERSONA_OWNER_AFACERE.md`
- `02_PERSONA_PROFESIONIST_JOB.md`
- `03_PERSONA_CONSULTANT_AI_AUTOMATION.md`

## De ce

- AI-ul citește mai ușor contextul când fișierele sunt scurte și clare.
- Git arată modificările mai bine în fișiere `.md` decât în documente mari.
- Echipa poate citi documentele `.docx`, iar AI-ul poate lucra pe `.md`.

## Reguli

- Un fișier = o idee principală.
- Nu amesteca persona, reclame și curriculum în același fișier AI.
- Păstrează un `MASTER_CONTEXT.md` actualizat.
- Păstrează `DECISION_LOG.md` pentru decizii importante.
- Nu pune API keys, parole sau date sensibile în repo.
